Program Development Tips

The procedures below provide general program design and development information using a typical academic environment approach. These instructions assume you use Java with an appropriate IDE or UNIX/Linux/Mac command line; however, most of these are adjustable to any language outside this course.

  1. Read the assigned problem statement and problem specification. These represent the exercise requirements with sufficient detail to indicate the problem to be solved (i.e., what you want to do) without specifying the solution (i.e., how to do it). It may require you to interpret the problem and decide how to approach it. In some cases, reasonable assumptions may be necessary. It is good to check with the instructor if your assumption is correct, but that is not always possible, so you should explicitly state it in your documentation. Plan to spend several days working on the problem, not just the due date.
  2. Identify the inputs and outputs of the problem. Although dangerous, you can create a skeleton program to read the data and write it out unchanged. It is risky because you have not yet designed a solution, and a skeleton program tempts you to continue coding rather than solve the problem first. Prepare sample data, or review the data supplied with the programming assignment. Study any examples provided to understand how the program transforms data.
  3. Design a solution to the problem and pseudocode/flowchart the design. Keep your mind to modularization using objects and methods/functions. You should provide an overview of functions and interactions, clearly identify the chosen data structures, and include individual module outlines in your documentation.
  4. Hand-trace some example data with your solution to see if you have a correct solution and have resolved as many issues as possible. If you are building your program incrementally, you should handle the significant cases first and then the error conditions.
  5. Prepare the program implementation using appropriate implementation techniques. Make sure you include the required documentation. Then execute the program for the indicated data sets (if there are no data sets, create your test data, but note that instructor-graded data may differ). Test with data extremes, minimums, and maximums; for example, if you must sort an array, run your sort on already sorted data and run it against the minimum- and maximum-size arrays.
  6. Check (verify) your results with the actual data provided. It often requires writing the results next to the original data and comparing them with your hand traces. Iterate steps 0-5 to obtain a complete, correctly working program on the target machine specified in class.
  7. Answer any required questions and perform any requested analysis using the results obtained in steps 4 and 5. You may need to run the program multiple times and use several data sets to complete the necessary analysis. Sometimes it may take time to run it on a dataset, so plan accordingly.
  8. Make sure you test your program by compiling and running it using the same software the grader will use. In our case, it will be Eclipse. Java should compile the same way in any IDE and run on any machine; however, you should check it because there are no guarantees.
  9. Submit your program and all necessary files as instructed. Ensure that all files are available for grading. It does not mean to ask the instructor if you have correctly submitted the assignment—that’s your job, and you can accomplish this by looking at what you submitted.
